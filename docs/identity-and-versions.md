# 设计说明:artifact 身份与版本语义

本文解释"什么时候递增版本、什么时候新建 URL"这一决策的设计。它是给维护者和 skill 编写者看的;Agent 需要知道的部分已浓缩进 `skill/open-artifacts/SKILL.md`。

## 背景:被观测到的失败模式

同一会话中先后两个任务:

> a. 介绍十二星座 → Agent 发布了一个 html artifact
> b. 现在再做一版幻灯片形式的 → Agent 发布了一个**新 URL** 的 slides artifact

用户期望 b 是 a 的新版本(同一 URL 演进),但 Agent 开了新链接。多种模型上可复现。

## 原因

失败不是模型的问题,是早期设计的三个缺陷叠加:

1. **身份键选错了受众。** 最初 artifact 身份完全由 `source_path`(发布文件的绝对路径)决定。路径对确定性脚本是好键,对 LLM 是坏键:它是 Agent 随手起的一次性名字,在上下文中毫无显著性,还会因 compaction / 跨会话而丢失。相反,URL 是打印给用户、双方都盯着的对象——它才是 LLM 容易复用的身份 token。
2. **slides 命名约定与"路径=身份"冲突。** slides 类型靠 `*.slides.html` 文件名触发,于是"把 html 改成幻灯片"最自然的动作就是换文件名 → 新路径 → 必然新 URL。b 轮的行为是被系统引导出来的。
3. **没有纠错回路。** 用新路径发布时脚本静默成功,Agent 甚至不知道自己刚做了一个"新建 vs 迭代"的决定。

根子上,"迭代还是全新"是真实的语义决策,但它被隐式化成了文件路径的副作用。

## 设计:三层防御

### 第一层:URL 是一等更新句柄(机制)

发布脚本支持 `--url <已发布链接>`:

```bash
publish.py xingzuo.slides.html --title "十二星座" --url http://127.0.0.1:8787/a/b3e84ab25a5d
```

- 带 `--url` = 对该 artifact 追加版本。**文件路径、文件名、内容类型都可以变**,URL 不变。
- 服务端同时把该 artifact 的 `source_path` **重绑定**到本次路径,此后对这个文件的裸路径发布(不带 `--url`)也会继续同一 artifact——两条身份线合流。
- 重绑定的边角:新路径已被其他 artifact 占用时,占用者被解绑(`source_path` 置 NULL,artifact 本身保留,可正常查看或删除)。这个情况恰恰出现在"误发新 URL 后按提示修复"的恢复流里,见下文。

身份优先级:**`--url` > 文件路径兜底 > 新建**。

### 第二层:发布输出自带"下一步模板"(关键巧劲)

每次发布成功,输出里都带着正确的下一步:

```
已创建: 十二星座 (v1)
http://127.0.0.1:8787/a/b3e84ab25a5d
注意: 这是一个新页面(新 URL)。若本意是迭代已有页面,应改用 --url <该页面链接> 重发。
迭代此页面(包括改文件名/改类型)时: 发布命令加 --url http://127.0.0.1:8787/a/b3e84ab25a5d
```

LLM 的强项是模仿上下文中的既有模式。a 轮的工具输出里躺着一条现成的更新命令,b 轮 Agent 照抄它的概率远高于从文档里回忆规则——**正确用法不依赖记忆,而是永远在场**。新建时的"注意"行则构成纠错信号:发错的瞬间,恢复路径就在眼前。

### 第三层:SKILL.md 写成决策规则(文档)

不再只是陈述语义,而是要求 Agent 发布前先过一个显式判断:

- **迭代**(同一会话中的默认):改内容、改形式(html ↔ slides ↔ md)、修问题、"再做一版/改成…" → 带 `--url`;
- **全新交付物**:新主题,或用户明确要求另开页面 → 不带 `--url`;
- 拿不准选迭代。

"改形式也是迭代"必须显式写出——这正是 b 轮的判断盲区。

## 误发新 URL 后的恢复流

1. b 轮误创建了新 artifact B(路径 `xingzuo.slides.html`);
2. Agent(或用户提醒后)用 `--url <A 的链接>` 重发同一文件;
3. 服务端把 A 的 `source_path` 重绑定为 `xingzuo.slides.html`,B 被解绑;
4. A 获得 slides 新版本,URL 不变;B 成为孤儿,可 `DELETE /api/artifacts/{B.id}` 清理。

## 与官方 Claude Code Artifacts 的对应

官方 `Artifact` 工具的语义与此同构:更新其他会话发布的 artifact 必须传 `url` 参数,否则"always mints a new URL";同一会话内同一文件路径重发则原地更新。本项目把"同一会话"的隐式状态替换为输出模板中的显式 `--url`,对多 Agent、多模型环境更稳。

## 验证要点(改动这部分代码时)

- `--url` 更新:版本 +1、URL 不变、类型可变、`source_path` 重绑定;
- 重绑定后裸路径发布续同一 artifact;
- 路径冲突时占用者被解绑且仍可访问;
- `--url` 指向不存在的 id → 404,脚本提示去掉 `--url`;
- 老库(`source_path NOT NULL`)首次启动自动迁移为可空。
