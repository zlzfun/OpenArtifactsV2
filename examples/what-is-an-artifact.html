Artifact:让会话的产出,活在一个链接里

终端擅长对话,不擅长呈现。Artifact 把一段会话的成果发布成 托管在固定 URL 上的活页面——会话继续,页面原地更新,链接始终是同一个。

三个词定义它

这三条合在一起,区分了 artifact 和"发个 HTML 文件给你"。

托管 hosted

发布后得到一个稳定 URL。接收者打开的是链接,不是附件;作者的画廊里能找到发过的每一个页面。

活的 live

每次重新发布都成为一个新版本,URL 不变。开着页面的人看到内容原地更新,也可以回看任意历史版本。

沙箱 sandboxed

内容在严格 CSP 的隔离 iframe 里渲染:不能发出任何外部请求。AI 生成的代码可以放心地跑在浏览器里。

同一个 URL,不断生长

artifact 的关键语义:重新发布同一个文件 = 同一链接的新版本。亲手试一下。

沙箱不是说说而已

这个页面正运行在 connect-src 'none' 的 CSP 下。两个按钮,一个能成一个必败:

可交互 · 真实的 CSP
// 结果会出现在这里
无外部请求
CDN 脚本、外部样式、字体、远程图片、fetch / XHR / WebSocket 一律被拦截。一切资源必须内联,图片用 data URI。
无后端
纯静态页面:表单无处提交,查看时不能调 API。交互状态只活在页面里。
单页
相对链接不会解析,多段内容用页内锚点——就像这个页面这样。
有上限
渲染后不超过 16 MiB。内嵌位图是超限的头号原因,图表请用 SVG 手绘。

什么时候用,什么时候别用

判断标准只有一条:这个产出,是"看"比"读"更清楚吗?

走读一次改动

渲染 diff 并在边栏加注释,评审者在代码旁边读你的理由。

呈现一组数据

会话里已经拉到的数据,变成带趋势线和状态色的仪表盘。

并排比较方案

几个布局、API 形态或实现路线摆进同一页网格,一眼见高下。

跟踪长任务

任务边跑边重发页面,开着链接的人看着清单一项项被勾掉。

✕ 不是内部工具

没有后端:不能存表单、不能鉴权、不能多路由。要部署应用,请上自己的基础设施。

✕ 不是一段普通文本

一句话能说清的结论,直接在终端里说。为它排版是浪费。

本页由 OpenArtifacts 发布——一个复刻 Claude Code Artifacts 的自托管实现。 页面全部资源内联,零外部请求,在 sandbox iframe 中渲染; 你读到的每一条约束,此刻都约束着这个页面自己。
